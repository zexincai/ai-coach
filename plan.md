核心痛点在于：**规划难、执行散、复盘弱**。
核心价值在于：**自动化拆解、动态调整、数据驱动的自我优化**。

---

### 1. 产品定位与名称建议
*   **产品名称建议**：FlowPlan / 智周计划 / AI Weekly Pilot
*   **Slogan**：让每一周都清晰可见，让每一天都从容执行。
*   **核心理念**：Intent-to-Action（从意图到行动）的闭环管理。

### 2. 核心功能模块设计

我们将产品分为四个核心阶段：**周末规划（Planning）**、**每日执行（Execution）**、**每日复盘（Review）**、**周度进化（Evolution）**。

#### 模块一：智能周末规划引擎 (周五 20:00)
**触发机制**：每周五晚 8:00 推送通知：“准备好迎接下周了吗？请输入你的想法。”

**功能逻辑**：
1.  **意图采集**：
    *   用户通过语音或文本输入下周的核心目标、杂事、灵感（例如：“下周要完成项目A的原型图，还要读半本书，记得约牙医”）。
    *   用户标记优先级（P0-紧急重要, P1-重要不紧急, P2-琐事）。
2.  **LLM 任务拆解与排期**：
    *   **拆解**：LLM将模糊意图拆解为可执行的原子任务（Atomic Tasks）。例如，“完成原型图”拆解为“竞品分析”、“草图绘制”、“高保真设计”。
    *   **估算**：LLM根据任务复杂度预估耗时。
    *   **排程算法**：
        *   结合用户设定的“工作习惯”（如：上午专注深度工作，下午处理沟通）。
        *   将任务均匀分布到周一至周五，预留缓冲时间 Buffer。
        *   P0任务优先安排在精力最好的时间段。
3.  **生成下周日历视图**：
    *   输出一个可视化的周计划表，用户可进行微调（拖拽修改、删除、新增）。
    *   确认后，任务正式写入数据库。

#### 模块二：每日晨间唤醒 (每日 08:00)
**触发机制**：每天早上 8:00 推送今日清单。

**功能逻辑**：
1.  **动态今日清单生成**：
    *   从周计划中提取当日任务。
    *   **关键逻辑**：检查昨日未完成的任务（Roll-over Tasks），自动合并到今日列表顶部，并标记为“延期”。
    *   LLM 生成一句“今日寄语”或“重点提醒”，基于今日任务的重要性给予心理暗示。
2.  **展示形式**：
    *   卡片式推送：显示今日 Top 3 重点任务 + 其余次要任务。
    *   提供“一键开始”按钮，进入专注模式。

#### 模块三：日间执行与晚间复盘 (每日 22:30)
**触发机制**：每晚 10:30 推送：“今天过得怎么样？来做个简短复盘吧。”

**功能逻辑**：
1.  **进度标记**：
    *   用户勾选已完成任务。
    *   对于未完成任务，用户需选择原因标签（如：预估不足、突发干扰、拖延、任务太难）。
    *   支持快速备注：“为什么没做完？”
2.  **LLM 动态调整（Rolling Plan）**：
    *   **未完成任务处理**：LLM 分析未完成原因。
        *   若是“预估不足”，重新评估剩余工作量。
        *   若是“优先级低”，建议移至下周或归档。
        *   默认动作：将未完成任务自动顺延至明日，并重新平衡明日的工作负载（如果明日已满，则提示用户调整或加班）。
3.  **微总结生成**：
    *   LLM 根据今日完成情况和用户备注，生成一段简短的“今日成就总结”，给予正向反馈。

#### 模块四：周度进化报告 (下周一早上或周日晚上)
**触发机制**：基于整周数据，由 LLM 进行分析。

**功能逻辑**：
1.  **数据分析维度**：
    *   **完成率分析**：计划 vs 实际。
    *   **时间分布**：深度工作 vs 浅层工作的比例。
    *   **拖延模式识别**：哪类任务最容易延期？（例如：每次遇到“写文档”就延期）。
    *   **精力曲线**：用户在哪些天/时段效率最高。
2.  **LLM 诊断与建议**：
    *   **归因分析**：“你本周未完成的任务主要集中在周二和周三下午，且多为创造性工作。数据显示你在下午3点后效率下降20%。”
    *   **改进建议**：“建议下周将创造性任务移至上午，下午安排会议或行政事务。”
    *   **下周规划辅助**：基于本周的遗留问题和节奏，为下周的规划提供预填充建议。

### 3. 技术实现思路与架构

#### A. 核心技术栈
*   **LLM 模型**：推荐使用具备强逻辑推理和长上下文能力的模型（如 Qwen-Max, GPT-4o）。需要精细的 Prompt Engineering。
*   **后端**：Python (FastAPI/Django) 或 Node.js。
*   **数据库**：
    *   PostgreSQL：存储用户信息、任务结构、历史记录。
    *   Vector DB (可选)：存储历史复盘数据，用于RAG检索，让LLM更懂用户的长期习惯。
*   **消息推送**：集成微信模板消息、钉钉机器人、iOS/Android Push Notification 或 Telegram Bot。

#### B. 关键 Prompt 策略 (示例)

**1. 任务拆解 Prompt (周五)**
```markdown
Role: 你是资深项目管理专家。
Input: 用户输入的原始意图列表 {{user_intents}}，优先级 {{priorities}}。
Constraint: 
1. 将大目标拆解为耗时不超过2小时的原子任务。
2. 根据任务性质（深度/浅层）分配到合适的时间段。
3. 每天保留20%的Buffer时间。
4. 输出格式为 JSON，包含：task_id, title, description, estimated_time, day_of_week, time_slot, priority。
```

**2. 动态调整 Prompt (每日22:30)**
```markdown
Role: 你是敏捷教练。
Input: 今日未完成任务列表 {{unfinished_tasks}}，用户标注的原因 {{reasons}}，明日原有计划 {{tomorrow_plan}}。
Action:
1. 评估未完成任务是否适合明天做。
2. 如果明日计划已满，建议移除或推迟哪些低优先级任务。
3. 输出更新后的明日计划 JSON。
```

**3. 周度分析 Prompt (周末)**
```markdown
Role: 你是个人效能分析师。
Input: 本周所有任务的完成状态、延期原因、用户备注。
Output:
1. 本周效能评分 (1-10)。
2. 发现的一个主要问题模式。
3. 三条具体的、可执行的改进建议。
4. 语气要鼓励且客观。
```

#### C. 数据结构设计简略
*   `User`: ID, Preferences (work_hours, peak_energy_time).
*   `WeekPlan`: ID, User_ID, Start_Date, Status (Draft/Active/Closed).
*   `Task`: ID, WeekPlan_ID, Title, Description, Priority, Estimated_Duration, Scheduled_Date, Scheduled_Time, Status (Pending/Done/Skipped/RolledOver), Completion_Note, Delay_Reason.
*   `DailyReview`: ID, Date, User_ID, Summary_Text, Mood_Score.

### 4. 用户体验 (UX) 关键点

1.  **极简输入**：周五晚上的输入框要足够大，支持语音转文字，降低用户启动规划的阻力。
2.  **可控感**：LLM 生成的计划必须是“草稿”，用户拥有最终编辑权。不要让用户觉得被算法控制。
3.  **正向反馈**：在完成任务时，给予视觉上的愉悦感（如打勾动画、进度条增长）。复盘时多鼓励，少指责。
4.  **无缝流转**：昨日的未完成项今日自动出现，无需用户手动搬运，这是提升留存的关键。

### 5. 潜在风险与应对

*   **风险**：LLM 拆解的任务不符合用户实际能力（过难或过易）。
    *   **应对**：引入“学习期”。前两周允许用户手动调整任务耗时，系统记录用户的实际耗时与预估耗时的偏差系数，后续自动校准。
*   **风险**：推送打扰。
    *   **应对**：允许用户自定义推送时间；若用户连续3天忽略晚间复盘，暂停推送并发送一条关怀消息询问是否需要调整节奏。
*   **风险**：隐私问题。
    *   **应对**：明确告知用户数据仅用于个人分析，提供“本地化处理”选项或匿名化处理选项。

### 6. MVP (最小可行性产品) 开发路线

1.  **第一阶段**：实现周五输入 -> LLM拆解 -> 生成周计划表格。实现每日早上推送今日任务。
2.  **第二阶段**：实现晚间打卡 -> 标记完成/未完成 -> 未完成自动顺延至明日。
3.  **第三阶段**：实现周度分析报告，加入简单的数据统计图表。
4.  **第四阶段**：加入个性化学习算法，优化任务预估准确度。
